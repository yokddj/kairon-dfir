## Qué significa “soporte parcial”

Soporte parcial significa normalmente que:

- ya existe detector o normalizador básico
- los registros pueden entrar en Search/Timeline
- pero todavía no hay una capa semántica tan madura como la de EVTX

## Recomendación operativa hoy

Si quieres un caso “rico” para validar la plataforma, prioriza:

1. `EvtxECmd_Output.csv`
2. Prefetch parseado
3. RECmd parseado
4. LECmd / JLECmd parseados
5. MFTECmd y SRUM

## Limitaciones actuales

- EVTX, Prefetch, LNK, Jump Lists y Registry ya tienen parsers específicos y ruta semántica propia.
- MFT y USN ya tienen parser específico, pero la correlación todavía es básica y no sustituye a una revisión cronológica manual.
- Recycle Bin ya tiene parser específico para `RBCmd` y `$I` raw, pero sigue interpretándose con prudencia: un archivo en papelera significa “enviado a la papelera”, no borrado permanente ni ejecución confirmada.
- Amcache y ShimCache ya tienen parser específico, pero se interpretan de forma conservadora: presencia o posible ejecución, no ejecución confirmada por defecto.
- PowerShell fuera de EVTX ya tiene parser específico para `PSReadLine`, transcripts y scripts observados, pero debe leerse con prudencia: historial y scripts no prueban por sí solos ejecución exitosa.
- USB enriquecido ya tiene parser específico para `setupapi.dev.log` y correlación con otros artefactos, pero USB observado no equivale a copia ni exfiltración confirmada.
- BITS ya tiene parser específico para CSV/JSON/TXT parseado y correlación con PowerShell/Browser/Defender/MFT/Prefetch, pero `qmgr` raw sigue en discovery-only hasta tener parser binario real.
- WMI ya tiene parser específico para CSV/JSON y clasificación de WMI Activity, pero `OBJECTS.DATA` / `INDEX.BTR` / `MAPPING*.MAP` siguen en discovery-only hasta tener parser binario real.
- Autoruns / ASEP ya tiene parser específico para CSV/TSV/XML y startup folder raw, pero los hives raw siguen dependiendo de parsers especializados como RECmd y no deben interpretarse como entradas parseadas por sí solos.
- Cloud Sync ya tiene parser genérico para CSV/JSON/logs pequeños y path inference sobre rutas cloud, pero no debe interpretarse como subida confirmada si no hay evidencia explícita de sync/upload.
- Network / WLAN / DNS ya tiene parser específico para WLAN XML, `hosts`, DNS/network CSV/JSON/TXT y clasificación de WLAN EVTX / Registry outputs parseados, pero los hives raw de red siguen en discovery-only hasta tener parser raw más profundo.
- La documentación de cada artefacto debe revisarse cada vez que se amplíe su parser o normalizador.
