# Generic Demo Guide

This guide describes a neutral workflow for demonstrating the platform without bundling a named public challenge or real evidence.

1. Create a new case in demo or training mode.
2. Upload a small synthetic evidence package that you are allowed to share.
3. Run the recommended indexing profile.
4. Use Search, Artifact Views, Command History, Findings and Incident Timeline to build an investigation narrative.
5. If you have an answer key, import a validation matrix package and compare coverage only at the end of the demo.
6. Generate a report from reviewed findings and curated timeline items.

Use neutral fixture names such as `HOST-A`, `HOST-B`, `USER-A`, `sample.iso`, `script.ps1`, and `example.com` in public screenshots and training material.
